# 数据结构大作业：短视频推荐系统需求说明

## 项目背景

本项目实现一个类似抖音、快手的短视频用户行为分析与推荐系统。当前项目已经拥有视频元数据文件：

```text
data/bilibili_10w_pro.csv
```

该文件已经满足 F1 的主要数据基础，字段包括 `aid`、`bvid`、`title`、`category`、`author`、`duration`、`pubdate`、`view_count`、`favorite`、`coin`、`share`、`like`、`tag` 等。下一阶段重点不是重新造视频库，而是基于真实视频元数据模拟约 12000 名用户的点击、观看、点赞、收藏、投币、分享等行为，使行为数据能够支撑相似用户、推荐、热度预测、视频聚类和用户聚类。

数据结构大作业不是单一算法实验，而是对软件设计的综合训练。本项目需要把推荐系统问题抽象为视频、用户、行为、兴趣向量、倒排索引、相似度计算、时间序列和聚类等结构，并尽量自己实现底层算法，避免把核心能力完全交给现成库。

## 功能需求

| 编号 | 功能 | 当前口径 |
| --- | --- | --- |
| F1 | 数据构建 | 使用 `data/bilibili_10w_pro.csv` 作为不少于 10 万条的视频信息库。 |
| F2 | 行为模拟 | 生成约 12000 名用户，以及点击、观看、点赞、收藏、投币、分享等行为日志。 |
| F3 | 相似用户分析 | 针对特定用户，基于兴趣向量和行为相似度找出相似用户 Top-K。 |
| F4 | 视频推荐 | 基于用户历史、相似用户、标签倒排索引和视频质量分推荐相关视频。 |
| F5 | 热度预测 | 基于某视频按时间窗口统计的历史观看量，预测未来热度变化。 |
| F6 | 视频聚类 | 将具有相似观看用户群体或相似受众向量的视频聚为一类。 |
| F7 | 用户聚类 | 将具有相似观看兴趣的用户聚为一类。 |

## 行为数据必须满足的条件

- 行为不能完全随机，否则 F3、F4、F6、F7 无法得到稳定结果。
- 用户需要有明确兴趣画像，例如偏好某些主题、分类或标签。
- 视频需要有主题向量和质量分，高质量视频应获得更高基础曝光。
- 交互行为需要满足漏斗关系：点击后才有观看，观看比例较高才更可能点赞、收藏、投币、分享。
- 行为时间戳必须晚于视频发布时间 `pubdate`，并呈现发布初期爆发、后续衰减、少量二次爆发的时间规律。
- 生成数据需要保留“真实标签”，例如用户兴趣群体、视频主题簇，方便后续验证聚类效果。

## 实现语言建议

核心实现建议使用 **C++17**。

原因：

- 课程是数据结构大作业，C++ 更适合展示数组、链表、哈希表、堆、图、并查集、稀疏向量等底层结构。
- 可以用标准库容器作为基础存储，但相似度、Top-K、KMeans、倒排索引、滑动窗口预测等核心算法应自己实现。
- 不建议用 `pandas`、`sklearn`、`numpy` 直接完成核心算法，否则报告中很难体现“数据结构与算法”的训练价值。
- Python 可以作为辅助工具，用于快速检查 CSV、画图或生成报告图片，但不要作为核心算法黑箱。

如果团队更熟悉 Python，也可以用 Python 完成最小原型，但正式版本仍建议把核心算法落到 C++，这样答辩时更容易解释每个数据结构的作用和复杂度。

## 验收口径

- 视频规模达到或超过 100000 条。
- 用户规模达到约 12000 人，至少超过课程要求的 10000 人。
- 行为日志包含点击、观看时长、完播、点赞、收藏、投币、分享和时间戳。
- F3-F7 都能通过模拟行为数据得到可解释结果。
- 每个核心算法都能说明输入、输出、使用的数据结构、时间复杂度和为什么适合本课程项目。
